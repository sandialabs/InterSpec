# ExportSpecFileTool — manual test matrix

Companion to `target/testing/test_ExportSpecFile.cpp`, and to the `.n42` files in this
directory (which that test generates - see `ensure_synthetic_files()`; re-running the test
overwrites them).  The unit tests cover the pure logic (`applicable_export_options()` /
`generate_file_to_save()`); this matrix covers what they cannot: real files, the live GUI,
the download path, and the formats' actual writers.

Run the app, then **InterSpec → "Export File..."** for every case.

```bash
cd build_vscode && ./InterSpec --docroot . --http-address 127.0.0.1 --http-port 8081 -c ./data/config/wt_config_localweb.xml
```

## 1. The rules being tested (from the code — expectations below derive from these)

### 1.1 Format record capacity — `maxRecordsInCurrentSaveType()`

| Class | `max_records` | Formats |
|---|---|---|
| **N-record** | `uint16_t::max` | N42-2012, N42-2006, CSV, TXT, PCF, GR-130, GR-135 |
| **2-record** | 2 | HTML; QR-code **iff** `num_gamma_channels < 2075` |
| **1-record** | 1 | CHN, SPC (int/float/ascii), IAEA SPE, CNF, TKA; QR-code iff `channels >= 2075` |

**The 2075-channel QR boundary is a real test axis**: a 1024-channel NaI file gets a 2-record
QR; a 16k-channel HPGe file gets a 1-record QR. Same option set must behave differently.

### 1.2 Option availability — `applicable_export_options()`

| Option | Shown when |
|---|---|
| Disp. Foreground / Background / Secondary | file has >1 sample, file **is** that displayed spectrum, and not Fore+Back mode |
| All Samples / Custom Samples | file has >1 sample, and not Fore+Back mode |
| Filter Detectors | >1 detector has data in the *selected* samples |
| Sum to single record | `max_records >= 2` and >1 record selected |
| Sum Fore/Back/Sec to single record | `max_records > 2` and that type is selected (or Fore+Back mode) and it has >1 record |
| Background subtract | Disp. Background selected **and** (Disp. Foreground or Disp. Secondary) selected |
| Sum det. per sample | `max_records > 2`, >1 sample, some sample has >1 detector |
| Sum samples per det. | `max_records > 2`, some detector spans >1 sample |
| Remove InterSpec info | format is N42-2012 or N42-2006 |
| Remove GPS | file has GPS |
| Lossless | format is QR-code |
| **(HTML: none of the above)** | HTML exports displayed spectra as-is; all options hidden |
| Fore+Back checkbox (top-left) | **at construction only**: foreground loaded AND background or secondary is a *different file* |

### 1.3 Generation rules

- 1-record formats **force** summing (the "Sum to single record" box is hidden, not needed).
- 2-record formats force summing **only** if >2 sample-sets were selected (i.e. fore+back+sec).
- Background subtract consumes the displayed fore/back/sec — they are not also summed separately.
- Output sample numbers are **always renumbered 1..N**, and peak-map keys are remapped with them.
- Peaks carry to an output record **only on exact sample-set key match** (by design).
- A file with exactly 1 measurement and "Remove GPS" unchecked is returned **as-is** (fast path).

## 3. Standard checks (apply to every export)

Every downloaded file should be re-opened in InterSpec (or inspected as text for SPE/CSV/N42):

1. **Record count** matches what the format allows and the status message claims.
2. **Sample numbers start at 1** and are contiguous.
3. **Live time / real time** are right (a sum of N records = N × single, not 1× or 2×).
4. **Source types** preserved (Foreground/Background) where applicable.
5. **Peaks**: present when the exported sample set exactly matches the fit set; SPE contains a
   `$PEAK_INFO_CSV:` block; N42 round-trips peaks into the Peak Manager.
6. **Detector names** — only the ones you selected.
7. **Titles** preserved (or "Background" for all-background portal sums).
8. **No error popups**, and no new lines in `developer_errors.log` (`PERFORM_DEVELOPER_CHECKS` builds).
9. **Status message** (bottom of dialog) agrees with what actually came out.
10. **No stale options**: switch file/format back and forth — visibility must not depend on order.

## 6. Cross-cutting

| Case | Steps | Expected |
|---|---|---|
| **Undo** | change several options → Ctrl+Z | Each step reverts one change |
| **Redo** | undo, then Ctrl+Shift+Z | **Goes forward, not back again** (redo was broken; this is the fix) |
| **URL state round-trip** | set options → close/reopen via undo | All checkboxes restored, including "Sum samples per det." (its handler never used to run) |
| **State restore doesn't clobber** | restore a state with only Custom Samples set | "All Samples" is **not** force-checked over it |
| **File switching** | switch between two loaded files in the combo | Options refresh; no stale checkboxes from the previous file |
| **Format switching** | cycle N42 → SPE → QR → HTML → N42 | Visibility identical to opening fresh on that format (no order dependence) |
| **Specific-file mode** | Spectrum Files tab → export a file that isn't displayed | File select hidden; correct file exported; detector list fixed |
| **No file selected** | clear the file select | Export/QR buttons disabled |
| **Phone layout** | `?isphone=1`, open dialog | Options reachable; format menu usable |

## 7. Targeted regressions (this round's fixes)

Each of these was a real, reproducible bug — worth an explicit pass:

1. **Peak loss** — S3, display sample 8, export SPE → `$PEAK_INFO_CSV` present. *(Was: silently missing.)*
2. **Needless sum** — same export → live time 300 s and title "Sample 8". *(Was: summed lone record.)*
3. **Gapped samples** — open `gapped_samples_peaks_on_9.n42` → peaks still show in Peak Manager on the sample they were fit on. *(Was: renumbered away, exceptions.)*
4. **Back-sub double-count** — S7 → CHN → counts ≈ fore − back. *(Was: ≈2× fore.)*
5. **QR over-production** — S13, fore+back+sec displayed → QR generates. *(Was: "internal logic error".)*
6. **Detector filter hidden** — S5 → Filter Detectors is offered. *(Was: hidden, unusable.)*
7. **Redo** — *(Was: redo re-applied the undo state.)*
8. **Sum-Back tooltip** — hover "Sum Back to single record" → gets the *background* tooltip; "Sum Sec" has one too. *(Was: on the wrong widget / missing.)*

## 8. Known-limitation checks (should be *documented*, not fixed)

- Peaks carried onto **summed** records keep original amplitudes/continua — they will not match the
  summed data. (Only background-subtraction refits continua.)
- Peaks are dropped when the exported sample set differs from the fit set (exact-match by design).
- HTML export ignores every option.
